# Описание работы
1. Класс CustomMutex
Реализован на основе std::atomic<bool> и compare_exchange_weak

Метод lock(): пытается установить флаг locked в true, если он был false

Метод unlock(): сбрасывает флаг locked в false

Использует yield() при неудачных попытках захвата для снижения нагрузки на CPU

2. Интеграция в программу
Заменен std::mutex kitchen_lock на CustomMutex kitchen_lock

Заменены std::lock_guard<std::mutex> на явные вызовы lock()/unlock()

Сохранена вся логика синхронизации из оригинальной программы

3. Преимущества реализации
Взаимное исключение: Гарантирует, что только один поток может находиться в критической секции

Отсутствие голодания: Использует yield() для fairness

Производительность: compare_exchange_weak эффективнее busy-waiting

Портретность: Использует стандартные средства C++ без платформенно-зависимых вызовов
